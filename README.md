# HospitalManage 项目说明文档

## 快速指南
- 后端服务启动：`docker-compose up --build -d`，访问 `http://localhost:3000`。
- 管理员账号登录后台：`admin` / `admin123`。
- 小程序端预览：使用微信开发者工具打开 `app/` 目录。
- 数据库初始化：首次启动时会自动执行 `./sql/init.sql` 创建表与初始数据。
- RabbitMQ 本地启动：在 `docker-compose.yml` 中已配置 RabbitMQ 服务，访问管理 UI `http://localhost:15672`，默认账号 `guest` / `guest`。
- 订单消息队列：参考 `doc/mq-guidelines.md` 了解发布/订阅规范与错误处理建议。
- 调试建议与开发流程：参考 `doc/前后端集成注意事项.md` 了解常见问题与改进要点。
- 若需帮助，请联系项目维护者。
## 项目功能
- 医生管理：管理员可增删改查医生信息与排班。
- 预约挂号：用户可查看医生排班并预约挂号，支持选择日期与时间段。
- 订单管理：管理员可查看所有预约订单，用户可查看个人预约记录与状态。
- 消息通知：基于 RabbitMQ 实现订单创建与取消的异步通知处理。
- 数据库支持：使用 MySQL 存储医生、用户与订单数据，支持事务与并发控制。
- 本地开发环境：使用 Docker Compose 快速搭建后端服务与数据库环境。
- 前后端分离：后端提供 RESTful API，前端使用微信小程序进行交互。
- 日志与监控：记录关键操作日志，支持 RabbitMQ 管理插件监控消息队列状态。
- 安全与鉴权：管理员接口需登录鉴权，用户操作需验证身份。
- 可扩展性：设计考虑未来功能扩展与第三方系统对接需求。
## 目录结构
- `app/`：微信小程序前端代码。
- `backend/`：Node.js 后端服务代码。
    - `routes/`：Express 路由定义。
    - `controllers/`：请求处理逻辑。
    - `services/`：业务逻辑与数据库交互。
    - `mq/`：消息队列发布与订阅实现。
    - `config/`：配置文件与环境变量。
- `sql/`：数据库初始化脚本。
- `doc/`：项目相关文档与开发指南。
